# 非功能需求（NFR）

## NFR-000 架构原则约束（不可变）
- 业务流程中仅允许访问去中心化 RPC。
- 禁止请求任何中心化后端业务服务。
- 该约束不允许被需求变更覆盖。
- 后台 0 依赖。

## NFR-001 性能
- 无明确业务价值时，禁止自动强制刷新。
- 同类 RPC 请求应去重、节流或缓存。
- 交易轮询应具备退避与终止策略。

## NFR-002 稳定性
- 本地存储恢复必须按 key 独立容错。
- 上下文切换（链/账户/Safe）不能残留错误状态。

## NFR-003 安全
- 助记词/私钥在导入成功后不应长期驻留在输入状态。
- 失败信息不泄露敏感上下文。
- 私钥仅允许驻留在浏览器内存中，不得上传、不得服务端存储。

## NFR-004 可测试性
- 每个关键链路必须有单元测试。
- 主流程必须有 UI/E2E 自动化覆盖。

## NFR-005 可维护性
- 需求变更必须先更新文档。
- 代码实现应可追溯到需求与故事 ID。

## NFR-006 隐私与遥测
- 零遥测：禁止分析脚本、追踪像素、行为埋点上报。
- 零身份：不建设用户数据库，不做用户身份识别。

## NFR-007 部署与可移植性
- 前端可作为静态站点部署到 IPFS。
- 功能可在无中心化后端的环境中完整运行。
