# 术语表

- EOA：外部账户，由私钥直接控制。
- Safe：多签合约账户，通过阈值签名执行交易。
- Pending Tx：已提交但未确认的交易记录。
- Nonce：账户交易序号，用于顺序与防重放。
- Force Refresh：绕过节流的强制数据刷新。
- Soft Refresh：遵循节流策略的数据刷新。
- 业务请求：影响产品业务结果的数据读写请求（如余额、交易、签名、状态查询）。
- RPC-Only：业务请求仅允许访问去中心化节点 RPC 的架构约束。
- 后台 0 依赖：产品不依赖任何中心化后端业务服务。
- 零遥测：不采集埋点、不上报分析数据、不使用追踪像素。
- 原生 Safe 交互：直接调用 Safe 合约，不依赖第三方 dApp 中转。
