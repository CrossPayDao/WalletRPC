# 用户画像与关键场景

## 1. 核心用户画像
- 稳定币支付用户（核心）：主要持有并使用 USDT、USDC 等主流稳定币，关注可用性、确认效率与支付确定性。
- 资产操作用户：需要快速、安全地转账与查看状态。
- 技术型隐私用户：具备研发或工程背景，关注隐私与网络安全，能够主动检查 HTTP/RPC 请求并判断风险。
- 多签协作者：需要在 Safe 流程中提议、签名、执行。
- 产品维护者（PM）：需要通过文档管理需求，不下沉到代码细节。

## 2. 高价值场景
- 场景 A：新用户导入助记词后，立即看到多链地址与余额。
- 场景 B：用户发起转账后，能够持续看到明确状态并可追踪到浏览器。
- 场景 C：多签成员在同一 Safe 下完成提议、签名、执行闭环。
- 场景 E：用户在业务过程中仅与去中心化 RPC 交互，不依赖中心化后端服务。
- 场景 F：用户无需第三方 dApp，直接在钱包界面完成原生 Gnosis Safe 全流程操作。
- 场景 G：产品可直接托管到 IPFS，前端上线不依赖中心化后台。
- 场景 H：用户以 USDT/USDC 为主进行日常支付，在不同网络与 RPC 节点下保持一致、可解释的支付体验。
- 场景 I：技术型用户通过内置 Console 检查 HTTP/RPC 请求，确认无遥测、无后台依赖、请求边界符合预期。
