# P0 不可变原则：RPC-Only

## 1. 原则定义
- 本产品是纯去中心化架构产品。
- 后台 0 依赖：不建设、不依赖任何中心化后端业务服务。
- 除 UI 渲染所需静态资源与前端运行依赖外，业务流程中的外部数据请求只能发往去中心化节点 RPC 接口。
- 禁止在业务流程中请求任何中心化后端业务服务。

## 2. 强制要求
- 需求、方案、实现、测试必须满足本原则。
- 该原则不可被需求优先级、上线时间、运营活动或临时决策覆盖。
- 若出现冲突，以本原则为最终裁决标准。
- 产品必须支持随时部署到 IPFS。
- 产品必须支持随时切换任意可用 RPC 终端节点。
- 私钥仅允许驻留在浏览器内存中，不得上传、不得落盘到服务端。
- 产品必须保持零遥测：无分析脚本、无追踪像素、无用户数据库。
- 多签能力必须基于原生 Gnosis Safe 合约交互，不依赖第三方 dApp。

## 3. 判定规则
- 合规请求：链上读写、交易广播、交易状态查询、多签状态查询等 RPC 请求。
- 非合规请求：中心化账户 API、业务聚合 API、服务端转发 API、中心化风控决策 API、遥测/埋点上报 API。

## 4. 例外边界
- UI 渲染相关：静态资源加载、前端依赖包下载、浏览器基础能力调用。
- 以上例外不属于业务流程决策，不得承载业务数据处理逻辑。

## 5. 违例处理
- 文档评审发现违例：需求驳回，不进入开发。
- 代码评审发现违例：阻断合并与上线，必须整改。
- 线上发现违例：作为 P0 事故处理，优先级最高。
