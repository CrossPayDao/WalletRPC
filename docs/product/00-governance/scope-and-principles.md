# 产品范围与原则

## 0. P0 不可变原则（RPC-Only）
- 除 UI 渲染所需静态资源与前端运行依赖外，业务流程中的外部数据请求只能访问去中心化节点 RPC。
- 业务流程中禁止访问任何中心化后端业务服务。
- 该条款不可被需求、版本或临时决策修改。
- 后台 0 依赖，支持 IPFS 部署与 RPC 节点自由切换。
- 零遥测：无分析脚本、无追踪像素、无用户数据库。

## 1. 产品范围（In Scope）
- 多链钱包基础能力：导入、地址展示、余额与代币资产读取。
- 交易能力：原生转账、代币转账、交易状态追踪。
- 多签能力：Safe 跟踪、提议、签名、执行、成员治理。
- 网络能力：链配置管理、RPC 配置、区块浏览器跳转。
- 本地持久化：用户自定义网络、代币、Safe 跟踪信息。
- TRON 原生资源治理理财（手动执行）：质押（能量/带宽）、SR 投票、奖励领取、奖励再质押与再投票（支持分步与一键串行入口，但每步均需用户签名确认）。

## 2. 非范围（Out of Scope）
- 托管私钥与服务端代持。
- 自动化理财执行（自动复投、自动调仓、定时策略任务）。
- 收益预测与承诺型指标（APR/年化预测、保本/保收益文案）。
- 第三方理财聚合（如中心化策略服务或非 RPC-only 的协议聚合 API）。
- 跨链桥聚合与路由优化。
- 中心化账户体系（手机号/邮箱注册登录）。
- 任何中心化业务中台、风控中台、账户中台依赖。

## 3. 核心原则
- 安全优先：敏感信息最小驻留，失败默认安全。
- 可解释优先：用户看到的状态必须有明确意义。
- 一致性优先：同一业务概念在文档、代码、测试中定义一致。
- 低冗余请求：无明确价值的自动刷新应被禁止。
- 架构自治优先：所有业务能力以去中心化 RPC 为唯一外部依赖。
- 隐私优先：不识别用户身份，不采集行为画像，不做遥测上报。
